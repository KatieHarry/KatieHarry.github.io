Java Development
Go Fish Card game

This year, I decided to learn Java in my free time and build a card game project with it. I had previously worked on card games and loved the logic that they required to implement, so I tried to choose a game that would use that logic. Go Fish might not seem like a very stratigic game at first glance, but as soon as I started implementing the AI for the computer players, I quickly realized the depth of strategy that goes into it.

I included three difficulty settings and an option to face varying numbers of opponents. The easiest AI just selects cards to request at random, while the hard mode computers remember every card you or other computers have asked for and compares them to their own cards, looking for a match. It has many layers of decision making if it doesn't know for certain if you have a card it wants. It chooses depending on things like how many cards you have and how many cards the computer has of each card and how close it is to making a set





This Go Fish game if part of a larger game that involves a store for purchasing new card games with in game money, as well as save data for this currency and game inventory, but I was particularly proud of the Go Fish game for the level of depth that the AI has, causing me to even have a hard time beating them.
